== Cross-Reference Examples

Full xref syntax: `xref:version@component:module:page.adoc#fragment[Link text]`

=== Automatic link text

Empty brackets use the target page's title as link text.

* xref:midpoint/quickstart/index.adoc[]
* xref:midpoint/architecture/index.adoc[]
* xref:midpoint:ROOT:reference/index.adoc[]
* xref:support-4.9@midpoint:ROOT:reference/index.adoc[]

=== Anchor within the same page

Fragment only; links to a section on the current page.

* xref:#section-one[Section One]
* xref:#section-two[Section Two]
* xref:#code-example[Code Example]

=== Same component, same module — different page

Component and module omitted; path is relative to `pages/`.

* xref:index.adoc[Site home page]
* xref:playground/a/renamed-index.adoc[Playground — A]
* xref:playground/b/index.adoc[Playground — B]
* xref:midpoint/index.adoc[MidPoint overview]
* xref:midpoint/quickstart/index.adoc[MidPoint quickstart guide]
* xref:midpoint/architecture/index.adoc[MidPoint architecture]
* xref:midpoint/architecture/concepts/index.adoc[MidPoint architecture concepts]
* xref:midpoint/security/advisories/index.adoc[Security advisories]

=== Same component, same module — page with anchor

Page path plus a fragment targets a specific section.

* xref:midpoint/quickstart/index.adoc#introduction[Quickstart — Introduction]
* xref:midpoint/quickstart/index.adoc#prerequisites[Quickstart — Prerequisites]
* xref:midpoint/architecture/concepts/index.adoc#midpoint-architectural-concepts[Architecture concepts heading]

=== Same component, different module

Module name required when the component has more than one module.

[source]
----
xref:other-module:page.adoc[Link text]
----

=== Different component — latest version

Component and module specified; version omitted to resolve the latest.

* xref:midpoint:ROOT:reference/index.adoc[MidPoint reference (latest)]
* xref:midpoint:ROOT:reference/roles-policies/policies/certification/index.adoc[Access Certification (latest)]
* xref:midpoint:ROOT:reference/admin-gui/resource-wizard/object-type/credentials/index.adoc[Credentials wizard (latest)]

=== Different component — explicit version

`version@` pins the xref to a specific version; the string must match `version` in the target's `antora.yml`.

| Git branch | version | xref

| `Dev`

| xref:development@midpoint:ROOT:reference/index.adoc[MidPoint reference — Development]

| `v4.10`
| `support-4.10`
| xref:support-4.10@midpoint:ROOT:reference/index.adoc[MidPoint reference — 4.10]

| `v4.9`

| xref:support-4.9@midpoint:ROOT:reference/index.adoc[MidPoint reference — 4.9]

Pinned version combined with a fragment.

* xref:support-4.10@midpoint:ROOT:reference/roles-policies/policies/certification/index.adoc[Access Certification — 4.10]
* xref:development@midpoint:ROOT:reference/admin-gui/resource-wizard/object-type/credentials/index.adoc[Credentials wizard — Development]

== Include Directive Examples

Full syntax: `include::family$path/to/file[attrs]`

Source files used below:

* `partials/playground/snippet.adoc` — AsciiDoc partial with tagged regions
* `examples/playground/config-sample.xml` — XML config with tagged regions

=== Entire partial

Includes the whole file; no attributes needed.

include::partial$playground/snippet.adoc[]

=== First N lines

Includes lines 1 through 5.

include::partial$playground/snippet.adoc[lines=1..5]

=== Line range

Includes lines 6 through 8.

include::partial$playground/snippet.adoc[lines=6..8]

=== Line index to end of file

Includes from line 14 to the last line.

include::partial$playground/snippet.adoc[lines=14..-1]

=== Non-contiguous lines

Includes specific lines picked from different parts of the file.

include::partial$playground/snippet.adoc[lines=2;3;6]

=== Single tag

Includes only the content enclosed by the named tag markers.

include::partial$playground/snippet.adoc[tag=intro]

=== Multiple tags

Includes two named regions, concatenated in declaration order.

include::partial$playground/snippet.adoc[tags=intro;footer]

=== Source file — entire file

Includes the full XML file from the `example$` family.

[source,xml]
----
include::example$playground/config-sample.xml[]
----

=== Source file — line range

Includes lines 6–10, which cover the `<datasource>` element body.

[source,xml]
----
include::example$playground/config-sample.xml[lines=6..10]
----

=== Source file — tag

Includes the `repository` block using XML-style tag comments.

[source,xml]
----
include::example$playground/config-sample.xml[tag=repository]
----

=== leveloffset

Shifts heading levels in the included content by +1.

[source]
----
\include::partial$playground/snippet.adoc[tag=intro,leveloffset=+1]
----

== Static Redirects

Antora generates a redirect page for every path listed in a page's `:page-aliases:` attribute.
By default the redirect is an HTML page with a `<meta http-equiv="refresh">` tag; Antora extensions can produce other forms such as `.htaccess` rewrite rules.

`playground/b/index.adoc` was moved from `playground/a/index.adoc` and declares the old path as an alias:

[source,asciidoc]
----
= This is the index page of `b`
:page-aliases: playground/a/index.adoc
----

Both xrefs below resolve to the same published page — the old path via the redirect, the new path directly:

* xref:playground/a/index.adoc[Link using the old path (alias)] — resolved via redirect
* xref:playground/b/index.adoc[Link using the new path (direct)]

== Attachment Examples

Downloadable binary resources (PDFs, ZIPs, etc.) stored in the `attachments/` family directory, referenced via `xref` with the `attachment$` coordinate.

=== Same component, same module

The `attachment$` coordinate is required; always provide link text as attachments have no default.

* xref:attachment$playground/midpoint-advanced-customization-sample-slides.pdf[Download the sample report (PDF)]
* xref:attachment$playground/sample-project.zip[Download the sample project (ZIP)]

=== No link text

Antora renders the raw resource ID as the link label when no link text is provided.

* xref:attachment$playground/midpoint-advanced-customization-sample-slides.pdf[]

=== Different component version

Full resource ID required when the attachment lives in another component version.

[source,asciidoc]
----
xref:support-4.9@midpoint:ROOT:attachment$midpoint-advanced-customization-sample-slides.pdf[Release notes — 4.9]
----

=== Different module, same component version

Module coordinate added when the attachment is in a different module.

[source,asciidoc]
----
xref:other-module:attachment$playground/midpoint-advanced-customization-sample-slides.pdf[Sample report from another module]
----


== Broken Links

Antora logs a build error for every xref or image reference that cannot be resolved.

=== Missing page

`playground/c/index.adoc` does not exist, so the xref below causes a build error:

* xref:playground/c/index.adoc[Link to a non-existent page]

[source]
----
ERROR: target of xref not found: playground/c/index.adoc
----

=== Missing image

`nonexistent.png` does not exist in the `images/` family, so the image below causes a build error:

image::nonexistent.png[Nonexistent image]

[source]
----
ERROR: target of image not found: nonexistent.png
----
